Privacy Policy

ImmoScout €/m² Badge (Chrome extension)
Last updated: 21 April 2026

This policy describes how the extension handles information. The extension is not affiliated with ImmoScout24 or ImmobilienScout24.

Single purpose

The extension has one purpose: on pages of https://www.immobilienscout24.de it reads price and living-area information that is already visible on listing cards, computes an approximate price per square metre (€/m²), and displays a small badge on listing images. Optional settings let you choose colours for different price bands.

Where the extension runs

Executable code runs only on www.immobilienscout24.de, as declared in the extension manifest. It does not inject scripts into other websites.

Data processing on your device

To compute €/m², the extension reads text from the page’s Document Object Model (DOM)—for example values shown in listing details—only in your browser. That processing happens locally to draw the badge. The extension does not send page content, browsing history, or badge calculations to the developer’s servers, because the extension does not operate a backend service for this functionality.

Storage (chrome.storage.sync)

Your colour preferences (thresholds, “above range” colour, and whether colours are enabled) are saved using the browser’s chrome.storage.sync API. If you use Chrome sync while signed into Google, Google may sync that small settings object across your devices according to Google’s privacy policy. The developer does not receive a copy of that data from the extension.

Remote code

All JavaScript runs from files packaged with the extension. The extension does not load or execute code from remote URLs, and does not use eval() on remote strings for its core behaviour.

Analytics and advertising

The extension does not include analytics SDKs, advertising, or tracking pixels controlled by the developer.

Children

The extension is not directed at children, and it does not knowingly collect personal information from children.

Changes

This policy may be updated when the extension changes. The “Last updated” date at the top will be revised accordingly.
